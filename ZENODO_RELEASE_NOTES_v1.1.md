# Zenodo v1.1 release checklist

1. Add the final clean manuscript and final Supporting Information files to `manuscript/`.
2. Confirm that `README.md`, `CITATION.cff`, and `zenodo_metadata.md` contain the correct GitHub URL and final DOI after release.
3. Commit all files to GitHub.
4. Create a GitHub release tagged `v1.1`.
5. Let Zenodo archive the release and generate a new DOI.
6. Replace the old DOI in the manuscript, SI, and Data Availability Statement with the DOI for the v1.1 release.
7. Resubmit the revised manuscript package to the journal.

Recommended release title:
`v1.1 - Peer-review revision reproducibility package`

Recommended release description:
`Revision package including updated dataset-flow documentation, validation-audit tables, feature-importance outputs, paired ranking effect sizes, corrected sanity-baseline/ranking outputs, and revised figures for the manuscript.`
